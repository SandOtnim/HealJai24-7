- ทำงานทุกหน้าผ่าน `common.js` — ทดสอบแล้ว: index เล่นถึงวินาทีที่ 3 → เข้าห้องพิพิธภัณฑ์เล่นต่อที่วินาทีที่ 3 ไม่รีเซ็ต
- ไฟล์ที่แก้: `assets/js/common.js`

**[หน้าแรก] แก้อินโทรเล่นซ้ำเวลากลับจากห้องมาหน้าแผนที่**
- เดิม: เข้าห้องเล็กแล้วกดกลับ → หน้าแผนที่เล่นอินโทร ("วันนี้คุณเหนื่อยกับอะไร…") ใหม่ทุกครั้ง
- แก้: อินโทรเล่นแค่ครั้งแรกของการเปิดเว็บ 1 รอบ (จำด้วย sessionStorage) — กลับจากห้องจะเข้าหน้าแผนที่/เกาะทันที
- ทดสอบแล้ว: เข้าครั้งแรก = อินโทรเล่น / กลับจากห้อง = แผนที่ขึ้นทันที
- ไฟล์ที่แก้: `assets/js/home.js`

**[ทั้งเว็บ] นำทางแบบไม่โหลดหน้าใหม่ (AJAX) → เพลงต่อเนื่องไร้รอย — ครบทุกห้องแล้ว**
- หลักการ: กดเข้าห้องแล้วดึงเนื้อหามาสลับในหน้าเดิม คง header + ตัวเล่นเพลง + กล่องบันทึกไว้ถาวร → เพลงเล่นต่อเนื่องไม่มี gap (ตัวเล่นไม่ถูกสร้างใหม่)
- ใช้ครบทั้ง 7 หน้า: หน้าแผนที่ + ห้องมู, ออกไปข้างนอก, พิพิธภัณฑ์, อยู่กับตัวเอง, ชุมชน, เกาะแห่งเวลา
- ทดสอบบนเซิร์ฟเวอร์จริง: เดินครบ 6 ห้อง เพลงไหลต่อเนื่อง 2.4→10 วิ ไม่รีเซ็ต, ไม่โหลดหน้าใหม่, ตัวเล่นเพลงตัวเดิม, ปุ่มย้อนกลับ (back) ทำงาน, ไม่เล่นอินโทรซ้ำ, 0 error
- เช็กว่าฟีเจอร์ในห้องยังกดได้หลังสลับ: ควิซห้องชุมชน (เริ่ม→คำถามขึ้น), พิพิธภัณฑ์ (exhibits + hotspot เปิด pane) ✓
- **ข้อควรรู้:** ทำงานเมื่อเว็บอยู่บนเซิร์ฟเวอร์จริง (เปิดผ่านลิงก์เว็บ/โฮสต์) เท่านั้น — ถ้าเปิดไฟล์ตรงๆ จากเครื่อง (double-click) จะถอยไปนำทางแบบปกติอัตโนมัติ (ไม่พัง แต่มี gap เหมือนเดิม)
- ไฟล์ใหม่: `assets/js/nav.js` | แก้: ทุกหน้า `.html` (เพิ่ม nav.js), `assets/js/home.js`, `assets/css/styles.css`
- หมายเหตุ production ในอนาคต: ควรเก็บกวาด event listener ของ home.js (tour/แผนที่) ที่สะสมเล็กน้อยเมื่อกลับเข้าหน้าแผนที่ซ้ำหลายรอบ (ตอนนี้ไม่กระทบการใช้งาน)

**[อยู่กับตัวเอง/ห้องนอน] จุดเช็คใจรายวัน → กราฟแนวโน้ม (ฟีเจอร์ใหม่)**

- ทดสอบแล้ว: บันทึก/อัปเดตรายวัน, กราฟ 7&30 วัน, prefill เมื่อเปิดใหม่, 0 error
